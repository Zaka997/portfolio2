Projet GLPI

Contexte :

Dans le cadre de mon projet, j'ai travaillé sur la gestion d'un parc informatique. L'objectif était de centraliser les informations sur le matériel et les logiciels, de suivre les incidents et les demandes d'assistance, et de faciliter la maintenance. Pour cela, nous avons utilisé GLPI, un logiciel libre spécialisé dans la gestion de parc informatique et le helpdesk.

Environnement :

L'environnement utilisé était basé sur Linux Debian 11. J'ai installé et configuré GLPI sur ce système, ce qui m'a permis de me familiariser avec l'administration sous Linux, la gestion de serveurs et les bases de données, puisque GLPI utilise MySQL/MariaDB pour stocker les informations.

Réalisation :

Durant ce projet, j'ai appris à ajouter et gérer des équipements comme les ordinateurs, imprimantes et périphériques. J'ai également suivi et traité des tickets d'incidents, attribué des matériels aux utilisateurs et mis à jour les historiques des interventions.

Bilan :

Cette expérience m'a permis de mieux comprendre le fonctionnement d'un service informatique et l'importance d'un outil centralisé pour la gestion des équipements et le suivi des incidents. (Concurrents : OCS Inventory, ManageEngine ServiceDesk Plus, Freshservice, Spiceworks, SysAid).

Application Mobile MAUI

Contexte :

Dans le cadre de mon deuxième stage de BTS SIO option SLAM, j'ai développé une application mobile Android de gestion de parking avec .NET MAUI. Ce projet avait pour double objectif : créer une interface mobile permettant de consulter en temps réel les parkings disponibles, les véhicules et leurs statuts, en communiquant avec une API REST développée en PHP, et également servir de support pédagogique sous forme de TP pour les étudiants de première année.

Environnement :

- Framework : .NET MAUI (Multi-platform App UI)
- Langage : C#
- Interface : XAML
- Communication : REST API avec HttpClient
- Plateforme cible : Android
- Outils : Visual Studio 2026, Émulateur Android, Git

Réalisation :

J'ai conçu une application avec trois pages principales : une page d'accueil affichant les statistiques en temps réel, une page listant tous les parkings avec leurs disponibilités, et une page présentant les véhicules avec un système de filtres dynamiques (toutes/garées/libres).

L'application utilise des appels HTTP asynchrones (async/await) pour communiquer avec l'API, désérialise automatiquement les réponses JSON en objets C#, et affiche les données dans des cards .

J'ai également géré les problématiques de compatibilité de types (conversion string → decimal pour les tarifs), configuré les permissions Android pour les appels réseau, et testé l'application sur émulateur et téléphone physique.

Bilan :

Ce projet m'a permis de maîtriser le développement mobile cross-platform avec .NET MAUI, la programmation asynchrone en C#, la consommation d'API REST, et la gestion des interfaces utilisateur avec XAML. Cette expérience a renforcé mes compétences en architecture logicielle et en communication client-serveur.

Projet Scolaire

Contexte :

Environnement :

- Langage : PHP / Laravel
- Base de données : MySQL
- Front-end : HTML, CSS, Bootstrap

Bilan :

Ce projet m'a permis de consolider mes compétences en développement web et de comprendre les avantages d'un framework structuré. J'ai également progressé dans la gestion de versions avec Git et la collaboration via GitHub.
